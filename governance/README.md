# Governance AI-SDLC Mob Programming Playbooks

## Purpose

This folder contains professional **SKILL.md governance playbooks** for applying AI-SDLC Mob Programming to architecture governance, security governance, risk management, compliance, policy-as-code, decision records, change governance, audit readiness, AI governance, and delivery governance.

These are not generic governance documents.

Each playbook is designed to help cross-functional teams run structured governance-focused mob sessions that produce decision-ready, implementation-ready, and audit-ready outcomes.

## How to Use These Playbooks

1. Open the relevant `SKILL.md`.
2. Use it as context inside GitHub Copilot Chat or another AI assistant.
3. Run a governance-focused mob session with the recommended roles.
4. Produce decision-ready artifacts.
5. Commit artifacts into the repository.
6. Convert follow-ups into GitHub Issues, Jira stories, Azure DevOps tasks, or governance actions.

---

## Core AI-SDLC Governance Question

```text
What do we need to decide, control, evidence, approve, monitor,
and govern before this capability is safe and ready for delivery?
```

---

## Expected Outputs

Across these playbooks, teams should produce:

- Architecture decision records
- Governance decision logs
- Risk registers
- Control mappings
- Compliance evidence lists
- Policy-as-code requirements
- Security review outputs
- AI usage guardrails
- Change approval artifacts
- Audit readiness checklists
- Delivery governance scorecards
- Exception registers
- Backlog-ready governance tasks

---

## Guiding Principle

Governance should not be a blocker discovered late in delivery.

In AI-SDLC Mob Programming, governance is designed collaboratively and early so that teams can deliver faster with clarity, accountability, and confidence.
