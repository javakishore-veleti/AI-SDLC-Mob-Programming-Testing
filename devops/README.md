# DevOps and SRE AI-SDLC Mob Programming Playbooks

## Purpose

This folder contains professional **SKILL.md playbooks** for applying AI-SDLC Mob Programming to DevOps, SRE, CI/CD, Cloud, Kubernetes, GitOps, DevSecOps, Release Management, Infrastructure-as-Code, and Platform Engineering.

These are not generic technology notes.

Each playbook is designed to help cross-functional teams run structured mob sessions that produce implementation-ready outcomes.

## How to Use These Playbooks

1. Open the relevant `SKILL.md`.
2. Use it as context inside GitHub Copilot Chat or your AI assistant.
3. Run a mob session with the recommended roles.
4. Produce the expected artifacts.
5. Commit those artifacts into the repository.
6. Convert implementation tasks into GitHub Issues, Jira stories, or Azure DevOps work items.

---

## Core AI-SDLC Mob Programming Question

```text
What do we need to understand, design, automate, test, secure,
monitor, operate, and recover before this DevOps capability is ready?
```

DevOps is not only automation.

SRE is not only monitoring.
